## 一、端侧语音交互(端侧离线语音控制,已合并主干)

给巡检机器人新增**完整的语音输入侧**,与已有语音播报(TTS)对称,全程端侧离线、不依赖外网。

### 1.1 交互链路
USB 麦克风 → **KWS 唤醒词**(「小巡」/「巡检助手」)→ **VAD 自动断句**(silero)→ **中文 ASR**(SenseVoice)→ **意图理解**(规则优先,听不懂再降级给 L1.5 端侧小模型解析成命令)→ **复用现有命令分发执行** + **TTS 语音回执**。一次唤醒内支持连续多轮,静默超时回到待唤醒。

### 1.2 研发过程(完整工程闭环)
- **设计文档 / 实现计划**:沉淀为 `docs/superpowers/specs/2026-06-21-voice-asr-interaction-design.md` 与对应实现计划;
- **实现**:拆成 **9 个任务**,每个任务独立测试驱动(TDD)+ 独立代码复审 + 修复循环;
- **最终全分支复审**:发现并修复一个**关键集成缺陷**(语音节点漏注册部分命令发布器,会导致复核/巡检/报告/验收类语音指令在真机报错)——逐任务测试因用桩件未暴露,靠全分支复审兜住;
- **合并主干**:分支快进合并到 `main`,合并后 **59 个单元测试**全绿。

### 1.3 交付清单(代码 + 配置 + 文档)
- **新增纯逻辑模块**(可开发机单测):`intent.py`(规则意图 + 小模型兜底解析)、`dialog.py`(唤醒应答/引导/移动类诚实降级话术)、`command_executor.py`(共享命令执行器 + 激光定时例程)、`asr_controller.py`(状态机 disabled/idle/dialog + 编排);
- **引擎与节点**:`asr_engine.py`(ASR 后端事件接口 + Mock + sherpa 骨架)、`asr_node.py`(rclpy 薄壳装配);
- **改造既有**:`command_receiver.py`(新增 `voice_control` 命令)、`command_receiver_node.py`(改为委托共享 `CommandExecutor`)、`qwen_client.py`(新增纯文本 chat 供意图兜底)、launch / setup 配线;
- **配置**:`config/asr.yaml`;
- **部署指南**:`docs/architecture/voice_asr_setup.md`(模型下载、麦克风探测、上板 bring-up、11 项验证清单)。

### 1.4 工程亮点
- **纯逻辑注入式架构**:意图/话术/状态机/命令执行都不依赖 ROS/推理库,可在开发机单测;rclpy 与 sherpa 只是薄壳。
- **命令通路统一**:抽出共享 `CommandExecutor`,让**语音指令**与**手机 App 下行指令**走同一套执行代码(复用巡检/复核/激光/验收/播报等既有能力),激光定时例程不重复。
- **诚实降级**:需要底盘移动的指令(到点复核/巡检),在底盘 PID 未完全打通前,语音回执如实说明"稍后执行",不假装完成。
- **算力可控**:唤醒/断句常驻极轻,识别/小模型按需触发;规则优先让 90%+ 常用指令不碰小模型。

### 1.5 远程开关与上板
- `voice_control` 命令:App 可一键开/关机器人语音监听(关闭即释放算力),状态本地持久化、重启保持;机器人侧接收执行 + 后端命令通道均已打通。
- 上板部署:离线安装 sherpa 运行时 + 三个模型(唤醒/断句/识别),麦克风走板载 ES8326;配置 **systemd 开机自启**,机器人上电后语音、云台、命令通道等全部自动拉起。

## 二、管理端 App 新增功能(Flutter,已出安装包)

在正式手机 App(`app/mobile`)上新增四项功能,均复用现有后端命令通道 + Bearer 鉴权。全部 `flutter analyze` 零问题、单元测试通过、**debug + release APK 均已出包(release 仅 53.9MB)**。

### 2.1 遥控驾驶页(安全攸关)
- 自绘**虚拟摇杆**(无第三方依赖):上下→前进/后退,左右→转向(可选横移模式);
- ~10Hz 把速度发给后端,机器人本地用**雷达实时避障**;
- **安全状态条**:畅通(绿)/减速(黄)/受阻(红)/离线(灰)+ 前方最近障碍距离;**急停 STOP** 大按钮;
- **安全底线 deadman**——松手 / 离开本页 / App 切后台 / 触摸中断,**任一情况立即发零速度**(且多发几次抗丢包),绝不让车卡着上一个速度继续跑;
- 速度严格夹在底盘限速内(前后 ±0.4m/s、转向 ±1.5rad/s);**19 个单元测试**覆盖摇杆→速度映射、越界夹紧、回零。

### 2.2 建图模式
- 一键进/退建图(机器人自动停语音栈腾资源 + 起建图栈);
- 开关状态**以机器人回报的真实模式为准**(2s 轮询):正常 / 切换中(转圈禁用)/ 建图中(绿)/ 进入失败(红,重试·退出)/ 离线(灰禁用);
- 建图中**内嵌遥控摇杆**开车绕图 + **一键存图**(地图名净化 `[A-Za-z0-9_-]`);App 重启后按真实状态还原;
- 工程上把遥控控件抽成可复用组件 `TeleopControlPanel`,遥控页与建图页共用,deadman 逻辑集中一处;新增"发命令 + 轮询回执"能力做结果提示。

### 2.3 播报音量
设置页音量滑块(0–100),拖动**释放时**下发 `set_volume`(避免拖动途中刷爆请求),乐观更新 + 本地持久化。

### 2.4 语音开关
设置页开关远程开/关机器人语音监听(`voice_control`),乐观更新,失败自动回滚,状态本地持久化。

### 2.5 契约修正(以真实后端代码为准,已回报)
- `set_volume`:按后端 API 契约 v1.3 实现(TTS 软件增益);
- 建图模式:文档写的 `POST /api/robot/commands` 系笔误,真实端点为 **`POST /api/commands`**(`set_mode`/`save_map` 已在命令白名单);
- `GET /api/robot/mode` 路由后端当前尚未挂载(仅有心跳存储预留)→ App 优雅降级为"离线"灰显,后端挂上即实时联动。

## 四、实验室现场实测(远程接入真机完成)

机器人带至实验室,经 **Tailscale 虚拟组网**远程接入(不依赖局域网),全程远程完成上电自检、接线排查修复与识别/融合验证。

### 4.1 连接与系统自检 —— 全绿
- Tailscale 可达、SSH 免密、机器人已联网(mDNS 的 `rdk.local` 跨网段不通,Tailscale IP 最稳);
- 开机自启的语音、命令接收、云台、激光、上行、认知等**节点全部在跑**;
- 底盘链路(STM32 USB CDC,`/dev/ttyACM0/1`)、云台编码器、IMU 均在线。

### 4.2 RGB 相机 + 目标识别 —— 正常
- MIPI 相机实拍 **1920×1072** 真帧(MIPI 相机走 hobot 视觉栈,不出现在 `/dev/video`,属正常);
- BPU 上的 YOLO 实时识别出危险物(线束、热风枪等),约 3–4 FPS。

### 4.3 热成像接线排查与修复(本次实测关键)
热成像模组(微雪 Thermal-90,红外辐射测温)线束**搬运易松**,本次连修两处:
- **出发前**:SPI 数据线(MISO/软片选)松 → 温度读数全为 −273℃(原始值全 0,无效)→ 重插修复;
- **到实验室**:电源 **VCC 断**(线太多没注意)→ I2C 地址不响应。**远程精准定位**:先排除软件侧(overlay 顺序、SPI 引脚 mux、SPI 传输在真实执行、I2C 控制路径均健康);再逐地址单读隔离——同一 I2C 总线上的**云台编码器(0x36)、IMU(0x68)均正常,仅热成像(0x40)掉线** → 判定为热成像模组自身供电断;接回 VCC 后温度恢复(**中心 ≈ 21.5℃ = 室温,数据逐帧刷新**)。
- **沉淀经验**(已记入项目记忆):该线束是反复故障点,明确"每次搬动先验、并用扎带固定两端"的现场流程;快速判据——温度全 −273 = SPI 数据线松、I2C 完全不响应 = 供电/数据线断(VCC 断会同时杀 I2C 与 SPI);排查序 VCC/GND → SDA/SCL → MISO/SS,杜邦线易内部断芯优先换新线。

### 4.4 识别能力 + 多模态融合验证
- 启动**生产融合检测器**(RGB 目标框 + 热成像热点 + 单应标定对齐),提供网页可视化预览(经 Tailscale 直接在浏览器实时看);
- **词表说明**:危险物模型 10 类(烙铁、焊台、热风枪、焊枪、电源排插、插头、适配器、线、线束、裸线);台式仪器(示波器等)不在词表内——**非漏检,是训练类别未覆盖**;
- **实测**:对当前仪器场景(示波器等),系统正确输出 **"无危险(objects=0)"——不误报**(对一堆仪器不乱打框,属正确行为);实时 ~3FPS;
- **融合逻辑验证**:模型已知的通电发热物体(如烙铁/热风枪)会触发"YOLO 目标框 + 热成像热点"融合 → `CRITICAL + 实测温度`红色横幅;无物体的热源 → "unidentified heat source";
- **结论**:识别与多模态融合链路端到端可用;现场演示只需放置模型已知的危险物(尤其通电发热的烙铁/热风枪)即可触发融合告警。
