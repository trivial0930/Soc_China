## A. 感知层:热成像 + RGB 融合(06-06 ~ 06-17)

**起点**:06-06 接手上一段对话(`rdk_x5_lab_detector_deploy_20260603`)的 L1 YOLO+热成像底座,目标"视觉+热成像多模态融合识别热源危险"。

- **Thermal-90(Waveshare)bring-up**:SPI `/dev/spidev5.0` + I2C `/dev/i2c-5`,SS 改 pin7;senxor 驱动集成 NUC(非均匀校正)+ 坏行修复 + 单帧读(`56389bce`)。
- **RGB↔热成像仿射标定**:真实仿射替换占位(`cd9904cf`);解决两画面互为镜像、需顺时针旋转 180°、`rotate_deg` 参数化(`b11fd418`/`ec8e99b2`)、检测框回原生相机系。
- **融合危险检测器**:`rgb_hazard_node` + 热成像融合,`publish_color=false` 解除检测节流(`7350fe9e`);逐物体标定(吹风机加热插排做热源实测)。
- **早期 bring-up 工作**累积提交在 `e8dc435d`(06-15 sync)。

**硬件/标定事实**:L1 = 板端 **YOLOv11s**(`hazard_yolo11s_640_nv12.bin`,10 类:烙铁/焊台/热风枪/焊枪/排插/插头/适配器/线/线束/裸线,BPU NV12 推理);RGB↔热成像单应标定写入 `thermal_rgb_calib.yaml`(热点质心↔YOLO 框中心,`cv2.findHomography` 3×3)。

**关键坑(记忆 [[thermal-90-rdk-bringup]] / [[thermal-transport-fragility-diagnostic]])**:热成像线束**搬运一次松一次**(SPI/MISO 或 VCC);i2c-5 被云台占总线会误报全空 → 用 smbus2 单读判 ACK;温度全 -273=SPI/MISO 松、I2C 不 ACK=VCC 断。

**实验室远程实测(06 下旬,经 Tailscale 远程,不到机器人跟前)**:
- **识别**:BPU YOLO 实时 ~3-4FPS;对仪器场景(示波器等,**不在词表**)正确输出"无危险 objects=0"**不误报**;
- **融合**:通电发热物(烙铁/热风枪)触发"YOLO 框 + 热成像热点" → `CRITICAL + 实测温度`红幅;无物体热源 → "unidentified heat source";
- **热成像线束 VCC 断远程定位**:同 I2C 总线云台 0x36/IMU 0x68 正常、仅热成像 0x40 掉线 → 判模组供电断;接回 VCC 温度恢复(中心 ≈ 21.5℃ = 室温)。

## 关键教训汇总

1. **USB 设备"冷启动失败/热插成功" = 上电时序/信号问题**(非线衰减、非软件缓存);看 dmesg `error -32`+low-speed 误判;同板其他 USB 正常就锁定该设备芯片。
2. **RDK X5 无可控 USB VBUS regulator**,纯软件切电走不通;必须软件切电只能外接 PPPS hub(uhubctl)。
3. **systemd 起 ROS 必须 `Environment=HOME=/root`**,否则找不到日志目录全崩。
4. **验 ROS 栈别信 topic hz**(best_effort/transient_local + DDS 发现假象),用 tf2_echo + node list + topic info 判。
5. **反复失联分两类**:黑药丸 USB 供电不稳(底盘链路)vs Mac 睡眠掐 gadget 以太网(ssh 链路)——根因不同别混。
6. **本地 VLM 选型**:thinking 焊死的模型(qwen3-vl:8b)对实时/L2 太慢;qwen2.5vl:7b 热调 1.4s 合适。
7. **仓库用 `~/projects/Soc_China`**(Desktop 被 macOS TCC 拦 EPERM)。
